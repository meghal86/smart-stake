üéØ Navigation Philosophy

Smart Back Navigation: The back button should take users to their previous screen, not a hardcoded destination. This respects the user's journey and provides predictable navigation behavior.

‚ùå Hardcoded Navigation

Problems:
  • Ignores user's actual journey
  • Breaks browser back button expectations
  • Forces users into predetermined paths
  • Confusing when coming from different entry points

‚úÖ Smart Back Navigation

Benefits:
  • Respects user's navigation history
  • Works from any entry point
  • Matches browser back button behavior
  • Intuitive and predictable

üîß Implementation Details

React Router navigate(-1):

  • Respects the actual navigation path the user took
  • Works with browser back button expectations
  • Handles edge cases gracefully (no history, direct access)

Code Changes:

  • WalletSettings: navigate(-1) instead of navigate('/cockpit')
  • AddWalletWizard: navigate(-1) for entry/exit screens
  • Success actions: navigate(-1) to return to origin
  • Button text: "Back" instead of "Home" or "Settings"
